# **Personal Finance Tracker**

## **Description:**

A personal finance tracking application using the MERN stack (MongoDB, Express, React, Node.js). The application should allow users to track their income and expenses, categorize them, and visualize their spending habits. The focus should be on building a user-friendly interface with data visualization.

## **Requirements**

### **Income and Expense Management:**

- **Add Transaction:** Users should be able to add income and expense transactions, including details such as amount, category, date, and description.
- **Edit/Delete Transaction:** Allow users to edit or delete existing transactions.
- **Categorization:** Transactions should be categorized (e.g., food, rent, salary, etc.), and users should be able to create custom categories.

### **Dashboard:**

- **Overview:** Display an overview of the user's financial status, including total income, total expenses, and current balance.
- **Category Breakdown:** Provide a breakdown of expenses by category.
- **Monthly Summary:** Display a monthly summary of income and expenses.

### **Data Visualization:**

- **Charts and Graphs:** Implement charts or graphs to visualize the user's spending habits over time. Use a charting library like Chart.js or D3.js.
- **Trends:** Highlight spending trends, such as categories with increasing expenses.

### **Error Handling:**

- Implement error handling for invalid input, network issues, and other potential problems.
- Provide clear feedback to users when errors occur.

### **Backend Requirements:**

- Create a RESTful API using Express and Node.js to manage transactions and user data.
- Use MongoDB to store transaction data and user information.

### **Frontend Requirements:**

- Use React to build the frontend of the application.
- Implement state management using either Context API or Redux.

### **Styling:**

- Focus on creating a clean and user-friendly interface. Basic styling is sufficient.

### **Technical Requirements:**

- Any third-party libraries can be used for data visualization.
- Any component library can be used or build your own components.

## **Additional Notes:**

- Ensure that the application is responsive and works well on mobile devices.
- Implement basic user authentication to secure user data.
